# 教研组资源共享一期方案（待审核）

- 方案版本：0.1
- 生成日期：2026-05-01
- 适用范围：教研组内资源共享、组内审核、广场准入
- 适用资源类型：实验列表、实验素材、实验材料库、实验课程、参考教材、实验题库
- 目标：以最小改造实现“教研组内共享 + 教研员审核后可进广场”的闭环

---

## 1. 方案目标

本一期方案的目标不是做复杂组织治理，也不是做多级协作系统，而是实现一个简单、清晰、可扩展的资源共享规则：

1. 普通成员上传的资源，默认仅自己和本教研组可见。
2. 同组的教研员默认是组管理员，拥有组内资源审核权限。
3. 教研员审核通过后，资源可以进入广场（公共池）。
4. 不做复杂权限矩阵，不做多组共享，不做资源版本协作。
5. 一期资源汇聚范围覆盖实验列表、实验素材、实验材料库、实验课程、参考教材、实验题库，不留二期。

---

## 2. 核心业务定义

### 2.1 教研组定位

教研组在一期中的定位是：

> **按学科划分的资源共享协作空间**

它的职责仅限于：

- 组内成员管理
- 组内资源共享
- 组内资源审核
- 资源进入广场的前置筛选

补充规则：

- 一个教研组只对应一个学科
- 同一用户可绑定多个学科
- 多个学科绑定对应多个教研组身份
- 教研员在本组默认 `ADMIN`，普通教师默认 `MEMBER`

### 2.2 教研组内角色

一期只保留两类组内身份：

- `ADMIN`：教研员默认即管理员
- `MEMBER`：普通成员

### 2.3 学科绑定与资源可见范围

一个教研组对应一个学科；教师和教研员通过学科进行绑定。

一期只保留两种基本可见范围：

- `PRIVATE`：个人可见
- `GROUP_ONLY`：个人 + 教研组可见

如果进入广场，再增加公共可见层：

- `SQUARE_VISIBLE`：广场可见

## 3. 资源流转规则

### 3.1 普通成员上传资源

普通成员上传教材、实验材料或实验列表后：

- 默认归属到个人空间或指定教研组
- 若归属到教研组，则默认状态为 `GROUP_ONLY`
- 只有自己和本组成员可见
- 不能直接进入广场

### 3.2 教研员审核资源

同组教研员可以：

- 查看本组待审资源
- 审核通过
- 审核驳回
- 将通过资源推进到广场

### 3.3 进入广场

资源进入广场的规则：

- 仅限同组教研员审核通过后进入
- 普通成员不能直接发布到广场
- 教研组决定资源审核的权限边界

## 4. 第一版功能范围

### 4.1 需要实现

#### 资源侧

- 教材资源可选择归属个人或某个教研组
- 实验材料可选择归属个人或某个教研组
- 实验列表可选择归属个人或某个教研组
- 教研组资源列表可见
- 资源状态可见
- 教研员可审核资源
- 教研员审核通过后资源可进入广场

#### 教研组侧

- 教研组成员列表
- 教研组管理员默认由教研员担任
- 教研组内资源查看
- 教研组内资源审核
- 教研组与学科一对一绑定

### 4.2 暂不实现

- 多教研组共享同一资源
- 组内协作编辑
- 复杂版本管理
- 资源权限矩阵（如可复制、可编辑、可转发等细粒度控制）
- 资源公开发布的二级审批链路

---

## 5. 权限规则

### 5.1 组内权限

#### 教研员（管理员）

可：

- 查看本组所有资源
- 审核本组成员资源
- 管理本组成员
- 将通过的资源推进到广场

不可：

- 以普通成员身份绕过业务规则直接污染公共池

#### 普通成员

可：

- 上传资源
- 发布到本教研组
- 查看本组资源
- 发起进入广场的申请动作（若流程保留）

不可：

- 直接进入广场
- 直接审批自己提交的资源
- 管理成员

### 5.2 广场权限

一期建议采用以下规则：

- 普通成员上传的资源默认仅组内可见
- 同组教研员审核通过后，资源才可进入广场
- 广场侧资源视为公共资源

## 6. 状态定义

### 6.1 资源状态建议

- `DRAFT`：草稿
- `GROUP_ONLY`：仅组内可见
- `PENDING_SQUARE`：待教研员审核，准备进入广场
- `SQUARE_APPROVED`：已进入广场
- `SQUARE_REJECTED`：已驳回进入广场
- `DISABLED`：停用

### 6.2 状态流转

#### 普通成员

`DRAFT` → `GROUP_ONLY` → `PENDING_SQUARE`

#### 教研员

`PENDING_SQUARE` → `SQUARE_APPROVED`

或

`PENDING_SQUARE` → `SQUARE_REJECTED`

### 6.3 默认行为

- 成员上传资源后默认先进入 `GROUP_ONLY`
- 如果需要进广场，再提交审核
- 教研员审核结果决定是否进入 `SQUARE_APPROVED`

## 7. 页面交互方案

### 7.1 资源编辑页

在教材、实验材料等编辑页增加以下控件：

- 归属范围：
  - 个人
  - 教研组
- 若选教研组，则下拉选择具体教研组
- 组内状态提示：
  - 仅组内可见
  - 待审核进入广场
  - 已进入广场

### 7.2 教研组页

教研组页面一期只做最简展示：

- 我的教研组
- 我参与的教研组
- 当前组成员
- 组内共享资源列表
- 待审核资源列表（仅管理员可见）

### 7.3 审核页

教研员审核页展示：

- 待审资源列表
- 资源详情
- 通过 / 驳回按钮
- 驳回理由填写

建议字段：

- `group_id`
- `group_name`

- `status`
- `subject_id`（一个教研组只对应一个学科）

- `group_id`

  - `ADMIN`
  - `MEMBER`
- `status`
- `create_user_id`

## 9. 权限系统接入建议

本方案一期必须接入权限体系，避免越权。

### 9.1 前端权限

建议最少接入：

- `PAGE_my_research_groups_READ`
- `PAGE_my_research_groups_WRITE`
- `PAGE_review_research_groups_READ`
- `PAGE_review_research_groups_WRITE`

### 9.2 后端权限

后端接口应增加：

- 教研组资源列表：读权限校验
- 资源创建/编辑/提交：写权限校验
- 教研员审核：审核写权限校验
- 教研员角色隔离：只允许教研员审核
- 同组校验：审核和组内可见必须验证当前用户属于该组，且教研员为该组 ADMIN

## 10. 接口建议

### 10.1 资源归属接口

- 创建资源时，传入归属类型和归属教研组
- 编辑资源时，可修改归属范围
- 查询资源详情时返回可见范围与审核状态

### 10.2 教研组资源接口

- 查询某教研组资源列表
- 查询某教研组待审资源
- 审核通过
- 审核驳回

### 10.3 广场接口

- 查询广场资源列表
- 查询资源是否已进入广场
- 审核进入广场

## 11. 风险控制

### 11.1 普通成员越权发布到广场

应对：

- 后端强校验，禁止普通成员直接审批通过
- 前端隐藏广场相关按钮

### 11.2 组内资源被非本组成员查看

应对：

- 所有组内资源查询都必须校验 groupId 与成员关系
- 非成员返回 403 或空列表

### 11.3 教研员身份误用

应对：

- 教研员在组内默认管理员
- 但必须明确是“同组教研员”才有审核资格

## 12. 一期实施建议顺序

### Phase 0：数据与权限准备

- 确认资源表是否能扩展归属字段
- 确认教研组成员表可直接复用
- 确认实验列表资源模型是否已有对应表或字段
- 确认权限码接入方案

### Phase 1：文案统一

- 教研组页面命名统一为“我的教研组”
- 审核页统一为“教研组审核”
- 页面文案统一使用“教研组”

### Phase 2：资源归属与组内可见

- 教材、实验材料编辑页增加教研组选择
- 列表页展示归属范围
- 组内成员可见资源列表

### Phase 3：审核与广场

- 教研员审核资源
- 通过后可进广场
- 驳回后保留驳回意见

### Phase 4：验收

- 验证组内可见
- 验证教研员可审
- 验证普通成员不能直接进广场
- 验证广场资源只显示已审核通过内容

---

## 13. 本期裁定

本方案为一期最简版本，原则如下：

- 只做组内共享
- 教研员默认管理员
- 普通成员仅组内和自己可见
- 同组教研员审核后可进广场
- 不做复杂协作与权限矩阵

如审核通过，可进入代码实施阶段。
