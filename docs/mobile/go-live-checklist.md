# 移动端上线检查清单

> 适用范围：`/m` 移动端全链路上线前验收。
>
> 目标：在不接入真实 API 的前提下，验证登录、绑定、学段分流、视频播放、个人中心、设置、死循环防护与安全锁基础行为均符合预期。

---

## 1. 验收说明

### 1.1 验收原则
- 所有检查项均以静态页面与本地上下文为准，不依赖真实后端 API。
- 重点验证“页面行为是否正确、跳转是否正确、视觉分流是否正确、是否存在重定向死循环”。
- 如遇到仍需接后端的能力，统一记入“已知遗留问题”。

### 1.2 角色覆盖
- 学生端
- 家长端
- 老师端

### 1.3 重点页面
- `/m/login`
- `/m/bind/child`
- `/m`
- `/m/video/[id]`
- `/m/profile`
- `/m/settings`
- `/m/assistant`

---

## 2. 核心链路验证项

### 2.1 学生端

#### 登录
- [ ] 使用学生账号登录后，可正常进入 `/m`
- [ ] 登录后不会被错误重定向到绑定页
- [ ] 登录态刷新后，页面仍保持学生端身份

#### 视频
- [ ] 可正常进入视频页 `/m/video/[id]`
- [ ] 视频页能展示静态视频内容与步骤区域
- [ ] 视频相关按钮可见且可操作

#### 个人中心
- [ ] `/m/profile` 能正确展示学生端布局
- [ ] 学段为小学时，展示小学风格的统计与快捷入口
- [ ] 学段为中学时，展示中学风格的统计与快捷入口

#### 设置
- [ ] 个人中心可进入 `/m/settings`
- [ ] 设置页可正常打开，不出现角色错乱或跳转异常

---

### 2.2 家长端

#### 登录
- [ ] 使用家长账号登录后，可进入 `/m/login`
- [ ] 若未绑定孩子，登录后会进入 `/m/bind/child`
- [ ] 若已绑定孩子，登录后可正常进入 `/m`

#### 绑定
- [ ] `/m/bind/child` 可正常展示绑定状态与孩子列表
- [ ] 绑定完成后能够回到 `/m`
- [ ] 绑定成功后不会再次被踢回绑定页
- [ ] 绑定状态在本地上下文中可见且稳定

#### 视频
- [ ] 家长可从首页或孩子内容进入视频页
- [ ] 视频内容可根据当前孩子上下文正常展示

#### 个人中心
- [ ] `/m/profile` 能正确展示家长端布局
- [ ] 可切换当前孩子
- [ ] 切换孩子后，展示内容同步变化

#### 设置
- [ ] 个人中心内可进入 `/m/settings`
- [ ] 设置页在家长端可正常打开

---

### 2.3 老师端

#### 登录
- [ ] 使用老师账号登录后，可正常进入 `/m`
- [ ] 不会被错误重定向到绑定页
- [ ] 登录态刷新后，页面仍保持老师端身份

#### 视频
- [ ] 可正常进入视频页 `/m/video/[id]`
- [ ] 视频页在老师端能展示教师风格或高信息密度内容

#### 个人中心
- [ ] `/m/profile` 能正确展示老师端布局
- [ ] 可正常查看教师功能、待办事项等内容

#### 设置
- [ ] 个人中心内可进入 `/m/settings`
- [ ] 设置页在老师端可正常打开

---

## 3. 死循环防护验证

### 3.1 基本重定向
- [ ] 未登录访问 `/m/**` 时，统一跳转到 `/m/login`
- [ ] 家长未绑定访问 `/m/**` 时，统一跳转到 `/m/bind/child`
- [ ] 学生或老师访问 `/m/**` 时，不应进入绑定页

### 3.2 白名单验证
- [ ] `/m/login` 不参与绑定拦截
- [ ] `/m/bind/child` 不参与绑定拦截
- [ ] 白名单页面自身不应触发重复重定向

### 3.3 绑定完成回跳
- [ ] 家长在 `/m/bind/child` 完成绑定后，回到 `/m`
- [ ] 回到 `/m` 后不会再次跳回 `/m/bind/child`
- [ ] 绑定完成后上下文中的 `hasBinding` 已更新

### 3.4 异常链路验证
- [ ] 连续刷新 `/m/login` 不会形成循环
- [ ] 连续刷新 `/m/bind/child` 不会形成循环
- [ ] 从任意 `/m/**` 页面手动切换路径后，路由守卫仍稳定工作

---

## 4. 视频安全锁验证

> 说明：当前为静态验收，重点检查安全锁 UI、按钮状态和步骤切换逻辑是否具备基础防护。

### 4.1 基础展示
- [ ] 视频页存在步骤列表或步骤引导区
- [ ] 安全提示区域可见
- [ ] 相关按钮、进度或锁定状态有明确视觉反馈

### 4.2 锁定逻辑
- [ ] 在安全提示未完成前，下一步按钮不可直接越权
- [ ] 锁定状态下，界面有明确禁用样式或遮罩提示
- [ ] 安全提示结束后，按钮恢复可用

### 4.3 边界场景
- [ ] 快速连续点击下一步，不会跳过安全步骤
- [ ] 切后台再返回后，锁定状态不应丢失
- [ ] 音频或播报未完成时，不应解除锁定

---

## 5. 学段分流验证

### 5.1 统一来源
- [ ] 学段判断统一来自 `MobileContext`
- [ ] 页面中不再自行使用字符串 `includes()` 做学段猜测
- [ ] `getSchoolStage()` 可返回 `primary`、`middle` 或 `unknown`

### 5.2 小学分流
- [ ] 小学账号进入 `/m` 后，首页视觉更活泼、偏暖色
- [ ] 小学账号进入 `/m/profile` 后，个人中心呈现小学版统计与入口
- [ ] 小学账号进入 `/m/assistant` 后，显示“魔法实验室”引导区

### 5.3 中学分流
- [ ] 中学账号进入 `/m` 后，首页视觉更紧凑、偏冷色
- [ ] 中学账号进入 `/m/profile` 后，个人中心呈现中学版统计与入口
- [ ] 中学账号进入 `/m/assistant` 后，显示“AI 对话助手”引导区

### 5.4 unknown 回退
- [ ] 当学段无法识别时，页面能够安全回退到默认展示
- [ ] 不应出现空白页或崩溃

## 7. 已知遗留问题

> 以下问题属于当前版本已知限制，非上线阻断项，但需在发布说明中明确。

- [ ] 真实后端 API 尚未完全接入，部分数据仍为前端静态写死
- [ ] AI 魔法球二期能力尚未接入，当前仅保留引导文案与输入框骨架
- [ ] 视频推荐、步骤联动、安全播报部分仍可能使用静态样例数据
- [ ] 上传、作业、统计等部分链路可能仍为骨架或 Mock 实现
- [ ] 不同设备下的 Web Speech 支持情况存在兼容性差异，需后续补齐降级策略
- [ ] 若后端 JWT 字段未完全统一，可能需要在联调环境继续补齐 `has_binding` 与 `school_level_id`

## 8. 上线判定

### 可上线条件
- [ ] 学生、家长、老师三类账号能完成登录验证
- [ ] 家长绑定链路无死循环
- [ ] 视频页可访问，安全锁基础行为正常
- [ ] 学段分流已统一为 `MobileContext` 驱动
- [ ] 个人中心与设置页可正常进入
- [ ] 已知遗留问题已在发布说明中标注

### 不可上线条件
- [ ] 出现登录后循环跳转
- [ ] 家长绑定后无法回到首页
- [ ] 小学/中学页面串台
- [ ] 视频安全锁失效，能直接越权跳步
- [ ] 设置页无法访问
