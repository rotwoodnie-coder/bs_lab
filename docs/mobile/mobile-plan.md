一、产品定位与核心价值
科学小实验社区 是一个面向 K-12 学生、家长、老师的移动端实验学习平台。
•	学生：通过视频学习、动手实验、上传作品、参与挑战与小法庭，培养科学素养。
•	家长：协助孩子完成实验、上传成果，查看学习进度与安全设置。
•	老师：制作实验课程、发布作业任务、批阅学生作品、管理题库与小法庭仲裁。
核心闭环：内容生产（视频+步骤+题目）→ 作业发布 → 学生/家长协作完成 → 老师批阅统计 → 社区互动与小法庭自净。

四、核心功能模块详解（含审计优化）
4.1 视频广场
配置项	小学版	中学/家长/老师版
布局	单列大卡片（16:9封面）	双列瀑布流（平板三列）
顶部栏	仅标题	搜索框 + 语音输入 + 横向筛选胶囊
卡片信息	封面、标题、教师、点赞/收藏、作业角标	同上；老师端额外显示“引用次数”
推荐算法	趣味性70%、安全20%、简单步骤10%	中学：知识点60%、热门20%、探究性20%
家长：材料易得性+安全完整性
老师：教材版本匹配+班级完成率+使用次数
•	接口要求：后端需支持按角色和年龄段返回不同排序结果的视频列表。
4.2 视频播放器与互动步骤
•	时间轴联动：播放器 timeupdate 事件高亮对应步骤；点击步骤可跳转。
•	安全提醒（红线）：
o	当前步骤的安全提示自动朗读（speechSynthesis），朗读期间“下一步”按钮 禁用，并叠加半透明蒙层 + 倒计时提示“安全播报中…”。
o	不可跳过，播完后自动启用“下一步”。
•	步骤数据：由老师标注或 AI 切片生成，需教研员审核。
4.3 AI 魔法球
4.3.1 小学版：魔法实验室（游戏化场景）
•	横向滚动场景卡片（科学岛、厨房区、浴室区、花园区、魔法工坊）。
•	点击场景 → 展示该场景下实验图标 → 点击实验 → 实验准备页 → 图片化步骤引导。
•	步骤引导增强：
o	语音指令支持关键词匹配（“下”、“走”、“后”触发下一步）。
o	大面积点击热区（整个卡片区域可点击）。
o	振动反馈（可选）。
o	离线缓存优先：使用 IndexedDB 缓存核心场景图和安全图标，断网时仍能显示安全提醒。
4.3.2 中学/家长/老师版：对话式 AI
•	半屏抽屉或全屏聊天界面，支持文字/语音。
•	后端：大语言模型 + 实验知识库 RAG。
•	快速模板：推荐实验、安全提示、材料清单、发布任务（老师）、孩子进度（家长）等。
4.4 个人中心（角色专属）
角色	主要模块
学生	我的实验、作业/任务、我的作品、勋章积分、班级、设置
家长	我的孩子（切换孩子）、作业/任务（协助上传）、安全设置、材料助手、通知、设置
老师	班级管理、作业/任务（发布与统计）、实验库、审核中心（含小法庭待仲裁）、数据统计、设置
家长绑定增强：老师端收到绑定请求时应有 Push 通知，避免因审核延迟导致家长无法协助作业。
4.5 作业任务闭环
1.	老师发布：从实验库选择实验 → 设置截止时间（可选） → 关联题目（可选） → 发布到班级。
2.	学生接收：个人中心「作业/任务」列表 → 点击任务观看视频 → 完成步骤引导 → 动手实验 → 上传成果（照片/视频）。
3.	家长协助：家长端可代孩子上传，系统标记“家长代交”，成果仍归属学生。
4.	老师批阅：查看成果 → 简单评级（优秀/良好/合格/不合格）+ 可选评语。
5.	学生查看：看到评级和评语，任务状态变为“已批阅”。
技术要点：
•	上传组件：分片上传 + localStorage 持久化 uploadId 支持断点续传。
•	使用 Page Visibility API + NoSleep.js 尽力维持页面活跃。
•	提交后后端更新任务状态，并推送站内信通知。
4.6 题库与每日挑战
•	题目管理：老师创建选择题/判断题/填空题，可关联实验。一期暂免审，后期教研员审核。
•	每日挑战：学生首次访问时从题库抽取题目（按年级）并缓存当天题目，答对得积分。
•	统计（写入时聚合）：
o	学生提交答案时，同步（或异步）更新 class_quiz_stats 冗余表。
o	老师查询班级答题统计时直接读表，响应毫秒级。
4.7 实验小法庭（亮点模块，中学版启用）
4.7.1 争议触发
•	用户举报（≥3次）→ 进入候选池。
•	AI 自动检测（可选高置信度）→ 直接标记。
•	老师手动标记（从审核中心）。
4.7.2 陪审团选拔
•	资格：完成安全测试 + 信誉分 ≥70。
•	随机选取 20 人，与上传者非同班/同校（地域隔离），确保公正性。
•	投票期 48 小时，超时弃权。
4.7.3 学生投票界面
•	播放视频，可在时间轴上打点标注“此处操作不规范”，为老师仲裁提供证据。
•	投票选项：规范 / 不规范（可勾选预设原因或简写理由）。
4.7.4 仲裁与反馈
•	若规范票 > 60% → 自动通过。
•	若不规范票 ≥ 40% → 推送老师终裁。
•	老师裁决：通过 / 驳回下架 / 要求修改。
•	投票学生获知结果，判断正确者信誉分 +5，错误者 -5；低于 60 分失去投票资格。
4.7.5 数据实体
•	disputed_videos（争议视频主表）
•	court_votes（投票记录，含打点信息）
•	学生表扩展 credibility_score
4.7.6 融合点
•	视频广场：争议视频投票期间隐藏或标记“审核中”。
•	个人中心-我的作品：展示争议状态与仲裁结果。
•	审核中心（老师）：新增“小法庭待仲裁”标签页。
•	实验闯关：安全测试作为小法庭准入条件。

五、家长绑定与多孩子管理（审计强化版）
5.1 绑定流程
1.	家长进入 /m/bind/child（未绑定时强制停留）。
2.	扫码或输入绑定码（学生端个人中心展示）。
3.	提交申请 → 状态“待审核”；老师端实时收到 Push 通知。
4.	老师审核通过 → 家长可正常使用。
5.2 多孩子切换
•	个人中心顶部显示孩子头像列表，点击切换。
•	全局状态 currentChildId 变化，所有 API 请求头自动携带该 ID。
•	后端中间件验证 家长-孩子 绑定关系，防止数据串线。
5.3 安全与隐私
•	家长只能查看已绑定孩子的数据。
•	解绑后数据立即不可见。
•	老师可查看所有绑定关系，必要时强制解绑。
