# 实验材料开发 AI 审查清单

> 适用于 Cursor Agent 与人工 Review 的联合检查。

## A. 数据模型一致性

- [ ] 主表 + 3 子表 + N 维表是否齐全。
- [ ] 所有关联是否有明确外键或等效约束。
- [ ] 关键唯一约束是否防止重复关系数据。

## B. 隔离与权限

- [ ] 查询是否都包含 `tenant_id + app_id`。
- [ ] 列表和详情是否执行 scope 授权判定。
- [ ] `expires_at` 过期授权是否被正确过滤。

## C. 筛选与索引

- [ ] 左侧筛选字段（分类/学段/风险/收藏）均有索引支撑。
- [ ] 适用范围是否通过 `edu_path_matrix` 关联，不用 JSON 存学段。
- [ ] 关键词检索方案是否明确定义（FULLTEXT 或替代方案）。

## D. 资源挂载

- [ ] 资源是否统一关联 `edu_media_registry.id`。
- [ ] 视频槽位是否校验媒体类型 `VIDEO`。
- [ ] `slot_key` 是否使用规范枚举，避免自由字符串污染。

## E. 共享能力

- [ ] 是否支持 `GROUP/SCHOOL/CLASS/ROLE/USER` 等主体授权。
- [ ] 课题组、学校、班级共享是否有查询验证用例。
- [ ] 权限掩码是否覆盖查看/编辑/管理/分享动作。

## F. 变更质量

- [ ] 迁移脚本可重复执行（字典种子使用 upsert）。
- [ ] 文档与 SQL 字段命名一致。
- [ ] 新增接口具备参数校验与错误码规范。

## G. 发布与回滚

- [ ] 测试库、预发库、生产库有清晰执行顺序。
- [ ] 生产回滚策略是否避免直接 DROP 造成数据损失。
- [ ] 核心冒烟用例已执行并留痕。
