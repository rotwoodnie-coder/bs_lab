实验云平台 AI 智能体模块 – 方案概要设计
版本：1.0 | 2026-05-07 | 状态：方案讨论

1. 模块目标与范围
1.1 总体目标
利用本地部署的大语言模型、语音识别及视觉技术，为教师、学生、家长、审核员提供自动化、智能化的实验课程创建与使用体验，显著降低实验资源制作成本，提升教学互动质量。

1.2 业务范围
教师端：实验视频智能解析、步骤自动生成、材料匹配、安全提示、出题建议、教材匹配推荐。

学生端：实验视频关键步骤跳转、安全语音提醒、实验报告辅助、知识问答。

家长端：亲子实验指导、孩子进度简报。

审核端：实验内容完整性检测、风险标注、审核意见草稿。

2. 核心功能概要
功能模块	简述	优先级
视频智能切片与步骤生成	语音转文字 → LLM识别步骤边界 → 输出带时间戳的步骤描述与关键帧	P0
实验材料自动提取与匹配	从视频/文本抽取器材名词，与材料库匹配，生成清单	P1
安全智能标注	识别危险操作或注意事项，自动填入安全提示字段	P1
知识点问答自动生成	基于实验内容生成选择题/简答题，供课后测验使用	P2
教材章节自动匹配	向量检索与课标/教材章节对齐，推荐挂载位置	P2
实验完整性检查	检测步骤、材料、结果等是否缺失，给出改进建议	P1
跨实验知识图谱	抽取知识点实体与关系，支持推荐与关联查询	P3
角色化AI智能助手	根据教师/学生/家长/审核员身份，提供个性化对话与操作引导	P0

设计原则：

AI 能力以 松耦合 方式挂接到现有 V2 实验服务，通过 API 调用，不侵入原有业务逻辑。

长耗时任务（视频解析）采用 异步模式，教师可随时查看进度并应用结果。

所有 AI 生成内容 必须先经教师确认 才写入正式实验表。

4. 技术选型概要
能力	技术组件	部署方式	选型理由
语音转文字	Faster-Whisper (large-v3)	Docker，GPU	速度快，中文准确率高，支持时间戳
大语言模型（推理）	vLLM + Qwen2.5-14B-Instruct	Docker，GPU	高性能推理框架，模型参数量适中，显存友好
代码辅助（vibe coding）	Ollama + DeepSeek-Coder-6.7B	宿主机进程	轻量，低显存占用，与业务 AI 共享 GPU0
视觉识别（可选）	YOLOv8 + PaddleOCR	Docker，GPU	成熟物体检测与文字识别
任务调度	Celery + Redis	Docker，CPU	可靠的异步任务队列
角色化助手接口	Python FastAPI	Docker/宿主机	轻量，易于集成
向量检索（教材匹配）	BGE-large-zh + FAISS	CPU	轻量，无需 GPU
知识图谱存储	关系表 + JSON 字段	MySQL	初期避免引入新存储，降低复杂度
5. 硬件资源概要（双 RTX 3090）

7. 关键依赖与集成点
7.1 与现有系统的集成
实验编辑器：增加“AI 智能解析”按钮，展示建议步骤/材料，支持勾选回填。

实验详情页：为学生增加 AI 助手聊天窗口，根据当前实验内容回答问题。

审核工作台：嵌入 AI 检测报告，显示风险项与缺失项。

用户会话：通过 x-user-id 获取角色，决定 AI 助手的行为边界。

7.2 数据依赖
视频文件需上传至现有 S3 存储，AI 模块通过 URL 可访问。

材料主库 material_msg 用于匹配；若不存在，需教师手动补充。

教材章节表 data_coursebook_unit 用于教材匹配推荐。

7.3 外部服务
无强制外部 API 依赖（全部本地化），但初期可选用公有云 ASR/LLM 快速验证。

8. 非功能性需求概要
维度	指标
响应时间	AI 助手对话 ≤ 2 秒；视频解析异步，≤ 10 分钟（10分钟视频）
并发能力	同时支持 5 个视频解析任务；AI 助手支持 20 个并发对话
可靠性	任务失败自动重试 2 次；关键数据定期备份
安全性	所有 AI 生成内容需教师确认；角色权限严格控制；不向互联网暴露模型 API
可扩展性	支持添加更多模型（如视觉、多模态）；支持水平扩展推理节点
可维护性	模型服务采用 Docker 封装，支持热更新；日志集中收集
9. 风险与约束概要
风险	缓解措施
语音转写专业术语准确率不足	提供人工校正界面；可微调模型
步骤边界识别错误	前端允许拖拽调整时间轴
模型推理延迟可能影响体验	异步处理长任务；对话使用较小模型（或量化版本）
硬件故障导致服务中断	双卡冗余（业务模型可临时切换至 GPU0）；定期快照
数据隐私（学生语音等）	所有处理在本地完成，不上传云端
10. 小结
本方案概要设计给出了实验云平台 AI 智能体模块的功能蓝图、技术选型、硬件分配及实施路径。核心亮点是 角色化 AI 助手 与 视频自动切片，两者均可基于双 RTX 3090 本地部署，成本可控且技术成熟。建议以 MVP 版本快速验证效果，再逐步增强智能化程度。
