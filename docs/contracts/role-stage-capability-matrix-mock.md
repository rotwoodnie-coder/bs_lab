# 科学实验社区 · 角色 × 环节 × 能力矩阵（Mock / 查缺补漏）

> **文档性质**：在 [`management-sidebar-ia-closed-loop-mock.md`](./management-sidebar-ia-closed-loop-mock.md) 闭环划分之上，按 **学生 / 家长 / 教师 / 教研员 / 校管理员 / 区管理员** 说明各环节 **职责、应有功能、与仓库导航/RBAC 对齐情况**，并列出 **缺口**。  
> **与《袁飞-宝山区「科学实验社区」智能体简介3》**：该文件位于工作区外路径，本环境 **无法可靠打开**（仅见临时锁文件 `~$...docx`）。本文 **教育理念与产品句** 与仓库已冻结叙述对齐：陶行知「生活即教育、社会即学校、教学做合一」、探究链 **「想法—方法—做法」**、**小先生制**（互教互学与众裁）、运营中心 AI / 法庭 / 亲子报告等（见 [`science-community-console-mock-spec.md`](./science-community-console-mock-spec.md)）。若需与 Word **逐条对表**，请将正文粘贴或把副本放入 `docs/contracts/` 后追加一节「Word 条款对照表」。

## 1. 环节模型（与闭环一致）

| 环节代码 | 名称 | 简述 |
|----------|------|------|
| **S1** | 标准与治理 | 课标、领域、年级标准、实验目录、全区策略与数据 |
| **S2** | 内容与资源 | 数字素材、实验材料、仿真工程 |
| **S3** | 实验生命周期 | 台账、探究方案编辑、评审、上下架与权限 |
| **S4** | 教学执行 | 班级、作业下发、门户学习与闯关 |
| **S5** | 评价与反馈 | 报告批改、亲子/会话报告、成长足迹 |
| **S6** | 社区与众裁 | 动态、审核、小法庭、话题挑战；与 AI 预检/发布流水线衔接 |
| **S7** | 组织与运营 | 组织树、账号角色、通知部署、审计、菜单裁剪 |

RBAC 真源摘要：`frontend/src/lib/rbac/management-access.ts`（校管 **不可** `/researcher`、`/console/review`；教研员 **不可** `/console/system`；上架/下架 **教研员、区管、超管**；步骤与材料 JSON **教研员、区管、超管** 可编）。

## 2. 分角色总览（在平台中的主站位）

| 角色 | 典型站位 | 门户（`portal`） | 管理台（`management`）真源 |
|------|----------|------------------|----------------------------|
| **学生** | 学习侧主体 | `PORTAL_NAV_ITEMS`：首页、实验库、实验工坊、闯关、成长足迹 | 我的任务、闯关、成长足迹（轻管理台） |
| **家长** | 协同与知情 | 同上门户四项 | 我的任务（占位）、孩子进度 |
| **教师** | 班级教学与内容编排 | 同上门户 | 工作台、班级、作业、批改、实验管理、题库、社区与法庭、素材、材料 |
| **教研员** | 标准、评审、中台运营（无全校账号壳） | 同上门户 | 运营中心、平台管理（**默认通知**，无 `system/users`）、实验管理、素材、材料、领域、仿真、全区看板 |
| **校管理员** | 本校组织与用户、有限运营 | 同上门户 | 组织架构、用户管理、社区与法庭、运营中心、平台管理 |
| **区管理员** | 区域目录、账号、治理 | 同上门户 | 与教研员 **同一套区管菜单**，但「平台管理」默认落地 **`/console/system/users`** |
| **超级管理员**（可选） | 全模块 | 同上门户 | `nav-config` 合并多角色菜单 + 菜单配置、课标对齐 |

## 3. 矩阵：角色 × 环节（作用 · 应有功能 · 当前匹配 · 缺口）

### 3.1 学生（`STUDENT`）

| 环节 | 在社区中的作用 | 应有功能 | 当前平台匹配 | 缺口 / 建议 |
|------|----------------|----------|--------------|-------------|
| S1 | 标准接受者，不直接定标 | 按年级/任务看到「该做什么」 | 门户实验库、任务承接页 | 与课标能力点的 **学生可读解释**（规划） |
| S2 | 使用资源做探究 | 学案、视频、仿真实验入口 | `/resources`、实验详情 | 材料包「我家有没有」清单（若简介强调生活化，可加强） |
| S3 | 不编方案；可产生学习轨迹 | 领取任务版本、提交作品 | `/experiments`、`/student/experiment-challenge` | **作品中心 / 申辩 / 下架红点**（契约已述，路由待补） |
| S4 | **做实验** 的主执行者 | 闯关、任务、课堂同步 | 门户四项 + 管理台闯关 | 与教师作业的 **强绑定任务列表**（Mock 可加强一致性文案） |
| S5 | 被评价、看反馈 | 成绩、评语、报告片段 | `/student/footprints` | AI 维度「想法—方法—做法」的 **学生可读反馈卡**（规划） |
| S6 | 众裁参与者（小先生） | 浏览、点赞、参与众裁（适龄） | 教师经 `/teacher/social` 枢纽进法庭与 **实验圈动态（优秀案例库）** | **学生侧明确「社区/众裁」入口** 与简介「小先生制」对齐度仍不足 |
| S7 | 无 | — | 无 | — |

### 3.2 家长（`PARENT`）

| 环节 | 作用 | 应有功能 | 当前匹配 | 缺口 |
|------|------|----------|----------|------|
| S1–S3 | 弱参与定标与编排 | 知情、签字、准备材料（可选） | 弱 | **家长任务** 仍为 `placeholder`；与简介「家校共育」差距大 |
| S4 | 督促与陪伴 | 看任务进度、亲子实验 | **`/parent/lab`**（家庭实验室；旧 `/parent/child-progress` 重定向） | 与 **亲子报告 / 会话** 的显式入口（契约有 `parent-reports`；`/parent/sessions/[id]`） |
| S5 | 接收成果与评价 | 报告、排名、AI 摘要 | 孩子成长足迹间接相关 | **一键查看「发给家长的实验报告」生成物** 缺专页 |
| S6 | 申诉与协理（可选） | 争议知情 | 无专页 | 与法庭 **申诉流程** 的家长视图（规划） |
| S7 | 无 | — | 无 | — |

### 3.3 教师（`TEACHER`）

| 环节 | 作用 | 应有功能 | 当前匹配 | 缺口 |
|------|------|----------|----------|------|
| S1 | 执行课标与校本要求 | 发布时对齐课标、引用区本目录 | 无独立区本菜单；课标页在超管合并项 `/researcher/curriculum-standards` **教师侧栏无直达** | **编辑器内课标插件（P1）**；或教师只读课标速查 |
| S2 | 备课与资源组合 | 素材库、材料核对 | `/teacher/materials`、`/experimental-materials` | 与目录 **引用关系** 可视化（Mock） |
| S3 | **编剧本、管班级实例** | 台账 + 编辑器 + 提交评审；**课题组** 创建与维护 | `/experiment-manage` + `/teacher/experiment-editor`；**`/teacher/research-project-groups`** | 与区本 **目录锁定字段** 的冲突提示（规划）；课题组与台账 **显式关联**（规划） |
| S4 | **唯一纽带**：分班与下发 | 班级、作业、门户衔接 | `/class/home`、`/teacher/assignments` | 作业与实验版本 **绑定校验**（规划） |
| S5 | 评价闭环 | 批改、反馈家长 | `/teacher/reports` | 与运营 **报告模板** 字段对齐的批改侧预览 |
| S6 | 教学侧治理 | 平局终裁、协查 | `/teacher/social`、`/console/social/court` | 两入口 **心智说明** 已在 IA 文档；可补 onboarding 提示 |
| S7 | 一般无 | — | 无 | 若简介要求「校本菜单」需与 `菜单配置` 角色范围对齐（当前仅超管显式入口） |

### 3.4 教研员（`RESEARCHER`）

| 环节 | 作用 | 应有功能 | 当前匹配 | 缺口 |
|------|------|----------|----------|------|
| S1 | **定标、调策略** | 领域、课标、目录维护、AI 策略 | 领域、仿真、全区看板；运营中心 AI；**无** `/console/system/users` | 课标页路径在 `/researcher/...`，与 `curriculum-standards` id 在超管菜单；教研员 **主导航未单列课标**（可通过 URL 或后续加一项） |
| S2 | 资源规范与质检 | 与教师同路径的素材/材料 | 有 | 区本 **资源审核队列** 与教师上传分流（规划） |
| S3 | **评审与可编辑内容** | 评审、改步骤/材料、上下架 | `/console/review/experiments`、`experiment-manage`、编辑器权限 `canEditMaterialsAndStepContent` | 与 **实验目录** 双向同步规则（文档级待写） |
| S4 | 通常不代课 | 视政策可观测 | 无班级/作业侧栏 | 若需要「入校观课数据」只读看板（规划） |
| S5 | 质量监测 | 抽检报告、模板参数 | 运营报告模板页 | 与教师批改数据的 **抽样视图**（规划） |
| S6 | 中台治理 | 评价审核、动态运营、法庭队列 | Console ops 全组（校管除外部分叶子） | 已对齐 spec |
| S7 | **不参与** 全校账号壳 | 无 system 用户/角色/组织 | `isManagementPathAllowedForRole` 拦截 `/console/system` | ** intentional**；与简介「教研专业权」分离账号权一致 |
| **课题组（跨环节）** | **校验与管理** | 审核教师提交的 **教研课题组**（组织实体）、生效后抽检/暂停/归档 | `/console/review/project-groups`（运营中心 **课题组校验**）；契约见 [`research-project-group-mock.md`](./research-project-group-mock.md) | 与 **实验评审** 并存；教师侧「创建课题组」完整表单与 API 为后续迭代 |

### 3.5 校管理员（`SCHOOL_ADMIN`）

| 环节 | 作用 | 应有功能 | 当前匹配 | 缺口 |
|------|------|----------|----------|------|
| S1 | 本校落实区政策 | 只读或转发区通知 | 平台通知、组织根 | **本校课标执行仪表盘**（只读，规划） |
| S2 | 保障开课条件 | 材料与设备（若启用） | 主导航无材料库；可经 URL？ | 侧栏 **无** 实验素材/材料/管理；若简介要求校管验货，需 **加入口或只读副本** |
| S3 | 一般不编探究方案 | 不参与评审 | **禁止** `/console/review`、`/researcher` | 与简介对齐：专业事归教研 |
| S4 | 保障教学班与人 | 本校班级与用户 | 组织、用户 | 无 `/class/home` 教师侧栏等价物；**校管排课/分班** 若需要应单列（规划） |
| S5 | 弱 | 舆情与投诉入口 | 无专页 | 可选 |
| S6 | 本校安全与法庭 | 小法庭为主 | Console 社区组 **仅小法庭**；`/console/social/court` | 与教师 **终裁** 分工需在帮助文案写清 |
| S7 | **核心** | 组织、用户、通知、部署、审计 | `organizations`、`users`、platform 组 | 与区管 **数据范围** 过滤（Mock 未区隔租户） |

### 3.6 区管理员（`DISTRICT_ADMIN`）

| 环节 | 作用 | 应有功能 | 当前匹配 | 缺口 |
|------|------|----------|----------|------|
| S1 | **区域真源** | 目录、看板、领域、课标（若赋权） | 全区看板、领域、仿真、平台管理全链路 | 课标汇总页在超管合并；区管 **可加主导航「课标对齐」** 以免去 `/researcher` 路径歧义 |
| S2–S3 | 治理与版本 | 同教研员增强 + 账号 | `experiment-manage` 上下架、编辑内容权限 | 多校 **租户筛选器**（规划） |
| S4–S6 | 区域运营 | 同教研员 + system 全权限 | 与教研员共用菜单 + `system/users` 默认落地 | 与教研员 **菜单去重** 仅靠 href 差异，可在 UI 标注「区管」 |
| S7 | 平台级 | 用户、角色、组织、审计 | 有 | 菜单配置在超管；区管是否可配 **区本菜单**（产品决策） |

## 4. 与《简介》常见条款的「查缺补漏」汇总（不依赖 Word 原文）

以下基于仓库已写入的 **简介对齐句** 与契约缺口表归纳；待 Word 入库后可逐条打勾。

| 简介主题 | 期望行为 | 当前状态 | 建议优先级 |
|----------|----------|----------|------------|
| **想法—方法—做法** + AI 无感注入 | 学生活动流中分层提示与报告维度 | AI 策略页、报告模板骨架 **已有**；学生端分层呈现 **弱** | P1 |
| **小先生制 / 众裁** | 学生可参与适龄众裁 | 法庭、动态在运营中心；**学生入口与规则说明** 不足 | P1 |
| **生活即教育 / 家校** | 家长任务、亲子报告、家庭材料 | 家长占位任务、进度页 **偏薄**；亲子报告 **BFF/API 叙事有、端不全** | P0–P1 |
| **教学做合一** | 教师编—布置—评—反哺目录 | 教师链 **较完整**；**反哺目录** 自动化未做 | P2 |
| **区本标准 + 学校执行** | 租户隔离、校管只读本区政策 | Mock **少租户隔离** | P1 |
| **智能体支点** | 年级策略、预检、安全 | `console/ai/strategies` **有**；与 `ai-precheck` **未接 API** | 见 science-community 缺口表 |
| **教研课题组** | 教师创建组、选成员；教研员审核与生效后管理 | 教研员入口 **已对齐**；教师创建流 **待补** | P1：与 `research-project-group-mock.md` API 列一致 |
