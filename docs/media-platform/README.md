# 媒体资源中台文档导航

本目录用于沉淀媒体资源中台的架构、数据库、接口、前端交互与执行手册。  
建议团队按本文档给出的阅读顺序推进评审与实施。

---

## 1. 文档清单与用途

### `00-overview.md`

- 用途：项目总览、目标边界、核心术语、分阶段路线。
- 适合角色：产品、架构、后端、前端、测试。

### `01-architecture-and-domain.md`

- 用途：六层架构、领域规则、状态机、存储安全策略。
- 适合角色：架构、后端、DBA。

### `02-api-and-ui-spec.md`

- 用途：后端 API 约定与前端交互规范。
- 适合角色：后端、前端、测试。

### `03-database-dictionary.md`

- 用途：数据库字段字典（人工核对版），逐表解释设计意图。
- 适合角色：后端、DBA、测试。

### `04-db-review-checklist.md`

- 用途：数据库评审打勾清单，保证设计与实现一致。
- 适合角色：架构、后端、DBA、测试负责人。

### `05-sql-runbook.md`

- 用途：SQL 执行手册（执行顺序、验证、回滚、报错处理）。
- 适合角色：DBA、后端、运维。

### `06-thumbnail-cover-pipeline-final.md`

- 用途：视频 / 图片 `THUMB_SM` 衍生与 `THUMBNAIL` Job 的**冻结设计方案**（代次防竞态、Worker 分流、API 聚合、STALE 语义、错误码与运维监控、材料封面优先级、分步实施路线）。
- 适合角色：架构、后端、前端、运维、测试。

### `../../database/migrations/0004_media_platform_init.sql`

- 用途：媒体资源中台初始化脚本。
- 适合角色：DBA、后端。

### `../../database/migrations/0006_media_outbox_events_init.sql`

- 用途：Outbox 事件表增量脚本（P1 自动化治理链路）。
- 适合角色：DBA、后端。

## 2. 推荐阅读顺序（首次接入）

1. `00-overview.md`：先统一目标与范围。
2. `01-architecture-and-domain.md`：确认模型和规则。
3. `03-database-dictionary.md`：核对字段语义。
4. `04-db-review-checklist.md`：组织评审并打勾留痕。
5. `05-sql-runbook.md`：按手册执行测试库建表与验证。
6. `02-api-and-ui-spec.md`：进入前后端联调。
7. `06-thumbnail-cover-pipeline-final.md`：实施缩略图 / 封面全链路时作为单一真源（可与 `02` 交叉阅读）。

---

## 3. 推荐实施顺序（执行路径）

### 第一步：评审冻结

- 使用 `04-db-review-checklist.md` 完成数据库评审。
- 评审通过后冻结字段和索引方案。

### 第二步：测试库落地

- 按 `05-sql-runbook.md` 在测试库执行 `0004` 脚本。
- 完成结构核验和冒烟联调。

### 第三步：接口联调

- 后端按 `02-api-and-ui-spec.md` 提供最小可用接口。
- 前端完成资源中心、引用弹窗、切片弹窗联调。

### 第三步（并行）：缩略图 / 封面管线（可选专项）

- 按 `06-thumbnail-cover-pipeline-final.md` **§12 分步实施路线图** 从阶段 A 推进至 F；与媒体登记、材料详情联调穿插进行。

### 第四步：预发与生产发布

- 预发重复测试库流程。
- 生产发布按 runbook 值守并保留回滚路径。

---

## 4. 角色分工建议

- 架构负责人：审核模型边界、状态机、迁移策略。
- DBA：执行 SQL、核对索引和外键、保障回滚可行。
- 后端负责人：事务一致性、权限校验、异步回执实现。
- 前端负责人：状态展示一致性与异常交互兜底。
- 测试负责人：主链路与异常链路回归覆盖。
- 运维负责人：发布值守、监控告警、故障演练。

---

## 5. 变更维护约定

- 新增或修改数据库字段时，必须同步更新：
  - `03-database-dictionary.md`
  - `04-db-review-checklist.md`（如检查项受影响）
  - `05-sql-runbook.md`（如执行或回滚流程受影响）
- 文档更新优先于代码合并，避免“代码先行、文档滞后”。

---

## 6. 里程碑建议

- P0：完成建表、引用复用、权限与任务回执闭环。
- P1：完成事件驱动对账、孤儿资源治理、快照版本升级。
- P2：完成自动标签、自动切片与语义检索增强。
