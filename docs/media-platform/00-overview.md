# 媒体资源中台开发总览

## 1. 文档目标

本文档用于定义媒体资源中台的建设目标、边界、术语与落地范围，支撑后续数据库、后端接口、前端交互和运维治理的一致实现。

本方案重点解决以下问题：

- 一份素材在多个业务场景复用，不复制物理文件。
- 视频内容支持切片后在不同场景按不同起止时间播放。
- 资源可被不同应用模块复用，具备中台化嵌入能力。
- 在暂未建设完整用户体系时，资源系统可独立上线。

## 2. 设计原则

- 去重优先：物理资源基于内容哈希唯一化。
- 引用优先：业务通过引用关系使用素材，不直接依赖物理路径。
- 解耦优先：存储、身份、权限与业务侧解耦。
- 演进优先：先支持无用户体系运行，再平滑迁移到真实用户体系。
- 安全优先：统一通过临时授权地址或代理流访问素材。

## 3. 中台能力边界

### 3.1 负责范围

- 资源上传、去重、登记、引用、检索、删除回收。
- 图片/文档预览衍生资源管理。
- 视频切片管理与片段复用。
- 资源访问授权校验与安全输出。
- 异步任务回执、失败重试、状态可观测。

### 3.2 不负责范围

- 宿主系统账号体系与登录鉴权实现细节。
- 宿主系统业务对象主数据管理（例如实验主表）。
- 宿主系统组织架构与角色模型细节。

## 4. 多系统嵌入策略

系统必须支持多应用共用同一资源库，核心策略如下：

- 全核心表增加 `tenant_id` + `app_id` 作为隔离键。
- 资源库内部仅存 `owner_type` + `owner_key`，不直接关联某系统用户表。
- 存储驱动通过抽象接口统一接入，不在业务层写死云厂商 API。

## 5. 核心术语

- Asset：物理资源，唯一对应一份内容文件。
- Registry：资源登记，提供业务可见名称和归属信息。
- Reference：业务引用，将素材挂载到具体业务场景。
- Scope：权限范围，定义谁可查看/下载/编辑/分享。
- Derivative：衍生资源，如缩略图、转码文件、预览文件。
- Segment：视频片段，定义起止时间和片段类型。
- Job：异步任务，记录处理状态、错误与重试。
- Principal：统一主体标识，如 `USER:1001`、`ORG:school-1`。

## 6. 分阶段实施建议

### P0（必须）

- 数据模型初始化（资产、登记、引用、权限、片段、任务）。
- 上传与去重、引用挂载、基础检索、片段创建。
- 本地代理下载与云端临时地址策略。
- 异步任务状态回执与失败重试。

### P1（增强）

- 事件总线（Outbox）与异步消费者解耦。
- 孤儿资源监控与重分类流程。
- 引用快照版本更新策略。

### P2（智能化）

- 自动标签抽取（OCR/ASR/CV）。
- 自动切片候选推荐。
- 语义检索排序增强。
