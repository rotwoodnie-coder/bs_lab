# OCR 识别功能说明

## 目的

实验编辑页中的 OCR 识别能力用于从视频封面或视频截图中提取关键信息，帮助老师快速回填实验名称、适用年级和相关引用信息，减少手工录入成本。

## 适用场景

- 选择媒体库中的实验视频后，自动触发 OCR 识别
- 本地上传视频后，自动触发 OCR 识别
- 在视频预览区手动点击“识别视频文字”时，重新执行 OCR
- 当封面图可用时，优先基于封面图进行识别

## 功能范围

OCR 识别结果会用于以下内容：

- 实验名称自动回填
- 年级自动匹配
- 学科信息辅助识别
- OCR 结果文本展示

## 交互说明

### 1. 自动触发

当用户完成以下动作时，系统会自动触发 OCR：

- 从媒体库选择视频
- 上传本地视频

### 2. 手动触发

视频预览区提供“识别视频文字”按钮，用于：

- 重新识别当前视频
- 在自动识别失败或结果不完整时补充识别
- 用户主动刷新 OCR 结果

### 3. 结果展示

OCR 识别结果在视频下方以文本形式展示，保持界面简洁。

建议展示内容包括：

- 实验名称
- 学科
- 年级
- 学校
- 主讲人
- 系列
- 原始识别文本

## 数据流

1. 用户选择视频或上传视频
2. 前端获取视频地址或封面地址
3. 调用 OCR 接口
4. 接口返回结构化结果
5. 前端执行字段回填
6. OCR 结果渲染到视频下方文本区

## 字段回填规则

### 实验名称

- 如果返回 `title`，优先写入实验名称
- 如已有用户手动输入，可保留用户值或按产品策略提示确认

### 年级

- 如果返回 `grade`，尝试匹配年级字典项
- 匹配成功后自动设置年级 ID 和学段 ID
- 若无法匹配，仅保留 OCR 文本，不强制回填

### 学科

- 如果返回 `subject`，可用于辅助字段推荐
- 具体是否自动写入，建议按产品策略确认

## UI 规范

- OCR 区域应放在视频预览下方
- 仅使用文本区展示结果，不使用复杂引用卡片
- 不影响视频主视觉的沉浸感
- 保持视频区为页面视觉核心

## 与现有状态流的关系

该功能不应改动页面核心状态流与 actions 接口，只在以下位置接入：

- 视频选择回调
- 本地上传回调
- 手动 OCR 按钮
- OCR 结果渲染区

## 异常处理

- OCR 请求失败时，不阻塞视频播放
- 识别失败应展示友好的文本提示
- 允许用户重复触发 OCR

## 备注

后续如果要扩展视频截图 OCR，建议复用当前 OCR 接口，仅替换输入图源为视频截图，而不是直接对视频地址做图片识别。
