# 中小学试验平台
# 数据库最终开发前设计文档
## 架构定稿版

**版本**：V1.0  
**用途**：数据库最终设计冻结 / 开发实施依据  
**范围**：数据库结构、字段设计、外键关系、命名规范、枚举规范、性能字段、审计字段  
**说明**：本文档为开发前最终设计定稿，用于确认数据库进入实施阶段。本文档不包含 SQL 执行脚本，SQL 将在本设计文档冻结后单独生成。

---

## 一、文档目的

本文档用于在数据库正式进入开发实施前，对最终结构、字段、外键、枚举、审计、性能优化和业务边界进行冻结，确保：

1. 数据库结构稳定可落地
2. 字段语义统一
3. 命名统一
4. 关键逻辑闭环完整
5. 性能优化字段提前纳入，避免后续动大表
6. 作业、题库、互动、积分、组织人员等核心业务均可顺利实施

---

## 二、最终设计原则

### 2.1 严格按源文件建模
所有表结构均来源于已确认 Excel / CSV 内容。

### 2.2 只做必要修正
允许的修正包括：
- 拼写纠错
- 字段类型优化
- 审计字段补齐
- 逻辑缺口补齐
- 性能字段补齐

### 2.3 不做无边界扩展
本项目不引入无关平台化扩张，不额外抽象通用中间件模型。

### 2.4 先闭环，后扩展
所有结构必须优先保证项目主线业务闭环，再考虑增强。

---

## 三、最终冻结结论

以下事项已确认纳入最终开发版本：

1. 时间字段统一为 `DATETIME` / `TIMESTAMP`
2. 明显拼写错误统一纠正
3. 审计字段统一补齐
4. 作业快照机制冻结
5. `data_file.file_size`、`data_file.file_ext` 纳入
6. `sys_org.org_path` 纳入
7. `exp_question_select.question_id` 纳入
8. 倒赞功能保留，但仅在实验小法庭中展示
9. `sys_user` 与 `sys_org` 作为统一主数据边界
10. `material_msg` 与 `data_file` 职责分离
11. `exp_library` 与 `exp_msg` 模板 / 实例边界冻结

---

# 七、素材资源模块最终设计

## 7.1 `material_msg`
### 字段
- `material_id` varchar(32) PK — 主键id
- `material_name` varchar(60) NOT NULL — 材料名称
- `material_prop_id` varchar(32) FK → `data_material_prop.prop_id` — 材料属性id
- `material_type_id` varchar(32) FK → `data_material_type.type_id` — 材料分类id
- `material_num` varchar(60) — 建议用量（含单位，如 "500 毫升"；前端输入时分"用量数值"+"单位"两个字段，存储前合并为此字段）
- `main_pic_url` varchar(200) — 主图片URL
- `exp_purpose` varchar(200) — 试验用途
- `additional_comments` varchar(200) — 补充说明
- `comments` text — 备注/安全说明
- `status` varchar(32) — 状态：y 启用，n 停用
- `create_user_id` varchar(32) — 创建人id
- `create_time` datetime — 创建时间
- `update_user_id` varchar(32) — 最后更新人id
- `update_time` datetime — 最后更新时间
- `is_deleted` tinyint NOT NULL DEFAULT 0 — 逻辑删除：0 正常，1 已删除

### 索引
- `idx_material_msg_prop` (`material_prop_id`)
- `idx_material_msg_type` (`material_type_id`)

### 说明
试验材料主库，与 `data_file` 职责分离。`material_num` 由迁移 `0043_alter_material_msg_num_and_comments.sql` 从 `int` 改为 `varchar(60)`（支持含单位的混合输入）；`comments` 从 `varchar(200)` 改为 `text`（支持较长文本）。
