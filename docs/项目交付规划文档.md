# 项目交付规划文档

## 一、项目定位

本项目是一个面向多角色的实验教育平台，不是单纯的实验编辑器或内容管理系统，而是围绕“实验资源建设、课程设计、审核治理、学习使用、成果传播”形成完整闭环的平台。

### 核心业务主线

1. **教研员主线**
   - 维护标准实验列表
   - 维护高质量实验视频
   - 提供标准实验资产
   - 审核教师课程
   - 推荐优质实验内容

2. **教师主线**
   - 基于标准实验进行课程设计
   - 使用实验视频作为核心教学素材
   - 编排实验步骤、原理、安全提示、结果说明
   - 提交课程审核

3. **学生 / 家长主线**
   - 面向实验学习和家庭实验指导
   - 支持实验指导、过程记录和成果分享

## 二、当前开发进度判断

当前项目已经具备较完整的业务骨架，已经不是从零开始的原型阶段，而是进入了“产品化收口期”。

### 已具备的能力

- 实验课程列表与管理能力
- 教师实验编辑能力
- 富文本与富媒体输入能力
- 实验步骤、结果、安全提示等内容模块
- 数据落库与保存草稿能力
- 提交审核、通过、驳回、下架等工作流能力
- 基础的角色分层意识

### 当前最重要的工作

不是继续堆叠功能，而是：

- 收拢角色入口
- 重构工作台信息架构
- 统一页面语义和交互风格
- 强化视频主视觉
- 明确教师与教研员的职责边界
- 提升整个平台的商业成熟感

- 结果描述

## 五、当前页面重构建议

### 已确认方向

当前教师实验编辑页建议升级为：

- 全屏桌面工作台
- 左侧目录导航
- 中间主编辑区
- 右侧辅助指引区
- 顶部状态栏和完成度展示

### 重点原则

1. 视频成为主视觉资产
2. 核心内容前置，扩展内容后置
3. 教师工作台与教研员审核中心分离
4. 页面语义正式化，减少开发态痕迹
5. 折叠低频内容，保证主任务清晰

## 七、交付边界说明

本阶段交付重点不是“功能再多一点”，而是：

- 页面结构合理
- 角色入口清晰
- 工作流明确
- 数据落库通畅
- 视觉和交互达到可用的商业产品标准
